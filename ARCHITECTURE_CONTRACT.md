# ARCHITECTURE_CONTRACT.md
### StrandsOfCode — Практический архитектурный контракт

---

## Цель документа

Этот документ — рабочий ориентир, а не надзорный орган. Он помогает принимать архитектурные решения последовательно и не запутаться, когда проект разрастается. Нарушение правил не отменяет работу — оно создаёт долг, который нужно погасить в ближайшей версии.

---

## 1. Слои архитектуры

Проект строится по принципу **UI → Controller → Services → Utils**. Каждый слой знает только о слое ниже, но не о слое выше.

```
ui/
  main_window.py        ← отображение и ввод пользователя
  history_dialog.py     ← диалоги
  dialogs.py
  styles.py

controllers/
  convert_controller.py ← точка входа для конвертации; маршрутизирует вызовы

services/
  file_collector.py     ← сбор файлов
  converter_service.py  ← конвертация в TXT
  *_converter_service.py← конвертация в HTML/MD/JSON
  *_to_code_converter.py← обратная конвертация
  base_reverse_converter.py ← базовый класс для обратных конвертеров

models/
  conversion_options.py ← параметры операции (dataclass)
  conversion_result.py  ← результат операции (dataclass)

utils/
  file_utils.py         ← работа с файлами, кодировки
  cancellation.py       ← механизм отмены

config.py               ← константы и настройки
```

---

## 2. Правила для каждого слоя

### UI (main_window.py, dialogs)

**Задача UI:** показывать состояние и принимать ввод. Больше ничего.

✅ Разрешено:
- Читать значения виджетов и передавать их в контроллер
- Показывать результаты, полученные из контроллера
- Управлять видимостью/состоянием кнопок и прогресс-бара
- Запускать QThread (ConversionWorker) для фоновых операций

❌ Запрещено:
- Вызывать `FileCollector`, конкретные конвертеры или любые сервисы напрямую
- Реализовывать логику определения форматов, путей, расширений — это дело контроллера
- Хранить список файлов для конвертации как постоянное состояние (только для отображения)

**Лимит размера:** `main_window.py` должен стремиться к ≤800 строк. Когда файл перевалит за 1000 строк — это сигнал вынести что-то в отдельный класс. Цель на v2.0: ≤600 строк.

**ConversionWorker:** допустимо держать в `main_window.py` пока файл не перегружен. Когда придёт время чистки — переносится в `ui/conversion_worker.py`.

### ConvertController

**Задача:** маршрутизировать запрос от UI к нужному сервису, нормализовать результат.

✅ Разрешено:
- Определять тип конвертации (код→текст, текст→код, код→код, текст→текст)
- Вызывать нужный сервис
- Нормализовывать разные форматы ответов сервисов в единый dict
- Собирать файлы через `FileCollector`

❌ Запрещено:
- Содержать логику самой конвертации (regex, форматирование контента)
- Знать о деталях UI (виджеты, Qt-объекты)
- Дублировать логику сервисов

### Services

**Задача:** конкретная работа с файлами и контентом.

✅ Разрешено:
- Читать файлы через `read_file_safe` из `file_utils`
- Создавать выходные файлы
- Реализовывать конкретный алгоритм конвертации
- Возвращать `dict` с результатом в стандартном формате (см. п. 3)

❌ Запрещено:
- Знать о Qt, UI, виджетах
- Вызывать другие сервисы (сервис → сервис — плохая зависимость)
- Иметь собственный `FileCollector` — используется один, из контроллера

### Utils

Чистые функции без состояния. Любой модуль проекта может их использовать.

## 3. Стандартный формат результата сервиса

Все сервисы должны возвращать dict в этом формате (или `ConversionResult`, который контроллер нормализует):

```python
{
    'total_files': int,
    'converted_files': int,
    'failed_files': List[str],   # строки с путями
    'output_files': List[Path],
    'errors': List[str],
    # опционально:
    'combined_file': {'output_path': Path},
    'total_size': int,           # байты
    'duration': timedelta,
}
```

**Почему это важно:** контроллер и UI не должны угадывать, какой ключ вернул тот или иной сервис. Используй `result.setdefault(...)` в контроллере для безопасной нормализации.

## 4. Что не дублировать

Следующие вещи должны существовать в **одном месте**:

| Что | Где должно быть |
|---|---|
| Построение дерева проекта (`build_project_tree`) | `utils/tree_utils.py` (создать при рефакторинге) |
| Маппинг расширений → язык | `utils/format_utils.py` или `config.py` |
| Маппинг язык → расширение файла | `BaseReverseConverter._get_extension_for_language()` |
| Чтение файлов с определением кодировки | `utils/file_utils.py` → `read_file_safe` |
| Механизм отмены | `utils/cancellation.py` → `check_cancelled()` |
| Уникальное имя файла | `utils/file_utils.py` → `ensure_unique_filename` |

Когда эта функция встречается второй раз — она выносится в утилиту, а не копируется.

## 6. Что запрещено всегда (без исключений)

- Закомментированный код, оставленный «на всякий случай»

- Один и тот же класс, определённый в двух разных файлах
- Прямые вызовы Qt-объектов из сервисного слоя
- `from PyQt6 import ...` за пределами папки `ui/`

## 7. Баг-отчёт: известные проблемы текущей версии (v1.3)

Эти проблемы зафиксированы и должны быть исправлены в порядке приоритета:

**Критично (ломает функциональность):**
- `CodeHTMLParser.handle_starttag` — `attrs` это список кортежей, не словарей. Обратная конвертация HTML→код не работает.

**Высокий (дублирование):**
- `build_project_tree` / `_create_project_tree` скопирован в 4 файла
- `_get_extension_for_language` скопирован в 4 файла (должен быть в `BaseReverseConverter`)
- `collect_python_project` и `collect_js_project` идентичны

**Средний (качество кода):**
- `import` внутри методов в `main_window.py`
- `print()` вместо `logging`
- Несогласованный формат результата между `ConverterService` (возвращает `ConversionResult`) и остальными сервисами (возвращают `dict`)

---

## 8. Концептуальные части (не трогать до v1.5+)

Следующие части кода написаны как «задел на будущее» и намеренно упрощены. Не рефакторить их раньше времени:

- `CodeToCodeConverter` — regex-транслитерация Python↔JS↔TS. Сейчас это proof-of-concept. Реальная конвертация потребует AST и отдельной версии.
- `detect_language` в `BaseReverseConverter` — эвристика, достаточная для MVP.
- История конвертаций — демо-данные в `HistoryDialog`. Реальная загрузка уже есть в `MainWindow`.
