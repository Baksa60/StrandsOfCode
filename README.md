# StrandsOfCode

Современный конвертер кода с улучшенной архитектурой и поддержкой множественных форматов.

## Особенности

- **Уникальная концепция** - прядение кода в текстовые нити
- **Современный интерфейс** - тёмная/светлая тема, Drag & Drop, чистый дизайн
- **Чистая архитектура** - MVC паттерн, разделение ответственности, отсутствие дублирования
- **Множественные форматы** - TXT, HTML, Markdown, JSON конвертация
- **Поддержка Python** - .py файлы с определением проектов
- **Поддержка JavaScript** - .js, .jsx файлы с определением проектов
- **Поддержка TypeScript** - .ts, .tsx файлы с определением проектов
- **Гибкие источники** - файлы, папки, рекурсивный поиск, проекты
- **Drag & Drop** - перетаскивание файлов и папок с умным определением типа
- **Статистика** - информация о файлах, размерах, прогресс
- **История** - сохранение истории конвертаций с аналитикой
- **Метаданные** - заголовки, нумерация строк, информация о файлах
- **Подсветка синтаксиса** - в HTML и Markdown форматах
- **Умное сохранение** - выбор папки до конвертации, уникальные имена
- **Логирование** - системное логирование для отладки и мониторинга

## Быстрый старт

### Установка

```bash
# Клонируйте репозиторий
git clone https://github.com/yourusername/strandsofcode.git
cd strandsofcode

# Установите зависимости
pip install -r requirements.txt
```

### Запуск

```bash
python main.py
```

## 📖 Использование

1. **Выберите источник файлов:**
   - 🎯 **Drag & Drop** - перетащите файлы/папки в окно приложения
   - 📄 Один файл через диалог выбора
   - 📁 Несколько файлов  
   - 📂 Папка (без вложенных)
   - 📂 Папка (рекурсивно)
   - 🚀 Программный проект (Python/JS/TS)

2. **Настройте вывод:**
   - 📄 **Формат**: TXT, HTML, Markdown, JSON
   - 📝 **Метаданные**: добавление заголовков и нумерации строк
   - 🎨 **Подсветка синтаксиса** для HTML/Markdown
   - 💾 **Сохранение**: выбор папки и генерация имени

3. **Запустите конвертацию** и наслаждайтесь результатом

## 🏗️ Архитектура

```
StrandsOfCode/
├─ main.py                     # Точка входа
├─ version.py                  # Версия и метаданные
├─ ui/
│  ├─ main_window.py          # Главное окно (Drag & Drop, UI логика)
│  ├─ dialogs.py              # Диалоговые окна
│  └─ styles.py              # Стили тем и визуальные эффекты
├─ controllers/
│  └─ convert_controller.py   # Контроллер конвертации
├─ services/
│  ├─ file_collector.py       # Сбор файлов
│  ├─ converter_service.py    # Базовая конвертация в TXT
│  ├─ html_converter_service.py    # Конвертация в HTML + CSS
│  ├─ markdown_converter_service.py # Конвертация в Markdown
│  ├─ json_converter_service.py     # Конвертация в JSON
│  ├─ text_to_text_converter.py     # Универсальный текстовый конвертер
│  ├─ base_reverse_converter.py     # База для обратной конвертации
│  ├─ code_to_code_converter.py     # Конвертер между языками
│  ├─ txt_to_code_converter.py      # Восстановление кода из TXT
│  ├─ html_to_code_converter.py     # Восстановление кода из HTML
│  ├─ json_to_code_converter.py     # Восстановление кода из JSON
│  └─ markdown_to_code_converter.py # Восстановление кода из Markdown
├─ models/
│  ├─ conversion_options.py   # Опции конвертации
│  └─ conversion_result.py    # Результат конвертации
├─ utils/
│  ├─ file_utils.py           # Файловые утилиты
│  └─ cancellation.py         # Отмена операций
├─ config.py                  # Конфигурация и настройки
└── requirements.txt           # Зависимости
```

## 🧬 Философия StrandsOfCode

Каждая строка кода - это нить в огромном полотне программы. **StrandsOfCode** помогает вам:
- **Прясть** эти нити в удобный текстовый формат
- **Сплетать** разные файлы в единое полотно документации
- **Ткать** понятную структуру из хаоса проектов

## 🤝 Вклад в проект

### 📋 Архитектурный контракт

**Важно:** Перед внесением изменений обязательно ознакомьтесь с [**ARCHITECTURE_CONTRACT.md**](ARCHITECTURE_CONTRACT.md) - это единственный источник правды о структуре проекта.

**Ключевые принципы:**
- 🎯 **ConversionEngine** - единственный владелец конвертации
- 🧩 **FormatDefinition** - чистая логика без UI зависимостей  
- ⚡ **Декларативные эффекты** - без прямых side-effects
- 🎨 **Специализированные UI** - Text-based vs Code-based форматы
- ❌ **Запрещено дублирование** - старый код удаляется, не оборачивается

### 🔄 Процесс внесения изменений

1. **Изучите контракт** - ARCHITECTURE_CONTRACT.md обязателен к прочтению
2. **Создайте ветку** - `git checkout -b feature/AmazingFeature`
3. **Следуйте контракту** - не нарушайте архитектурные принципы
4. **Тестируйте** - убедитесь, что ничего не сломалось
5. **Создайте PR** - с описанием изменений и их соответствия контракту

### 🚫 Что запрещено

- Создавать дублирующие системы управления состоянием
- Добавлять формат-специфичные UI компоненты
- Хранить состояние в FormatDefinition
- Вызывать PyQt/dialogs из бизнес-логики
- Оставлять старый код "на всякий случай"

**Работа отклоняется**, даже если "всё работает", но нарушен контракт!
